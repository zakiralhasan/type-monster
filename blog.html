What is the difference between local storage and session storage?

The Storage interface of the Web Storage API provides access to a particular domain's session or local storage. It allows, for example, the addition, modification, or deletion of stored data items.

Local storage:

  • Local storage data has no expiration limit, even if the browser or session is closed.
  • It store key/value pairs for a particular domain.
  • The values stored may only be strings.

Session storage:

  • Session storage data is deleted after the browser or session is closed.
  • Session storage store the data key/value pairs for a particular domain.
  • The values stored may only be strings.

What is the difference between global scope and block scope?

Global scope:

A variable exists inside or outside a block. If a variable is declared outside all functions or curly braces ({}), it exists in the global scope. The global variables can be accessed by any line of code in the program, including inside blocks.

Block scope:

The variables that are declared within the block {} are called block scope. It cannot be accessible outside the block.

How javascript event loop work?

Event loop's working procedure is given below:

1. Any variable declared is first stored in the memory Heap.

2. When a synchronous function is declared, it is placed on the Call stack as LIFO (last in fast out).

3. When an asynchronous function is declared, it is placed on the Web API.

4. After finishing the asynchronous functions work, it is placed on the Callback Queue.

5. Then event loop takes an asynchronous function from the Callback Queue and placed it in the Call stack if the call stack is free or no synchronous function's work is pending.

Summary:Then event loop takes an asynchronous function from the Callback Queue and placed it in the Call stack if the call stack is free or no synchronous function's work is pending.

When we will find "undefined" as a value of any variable in javascript?

Some reasons for finding undefined as a value in Javascript are given below:

1. When declared any variable without value.

2. When called a function without declaration.

3. When trying to access a non-existing object property.

4. Without return statement, a JavaScript function returns undefined.

5. When trying to access an array element with an out-of-bounds index, etc.
